# 工程需求池

这份文档记录模型研究和运行时开发之间的工程契约。它不是排期表，只记录“为了让 War3-like 试玩变可信，工程需要接什么、验证什么”。

## 当前接入原则

- runtime key 必须以 `src/game/GameData.ts` 为准。
- 模型接入优先通过 `src/game/AssetCatalog.ts`，避免在 visual factory 里堆特殊路径。
- 团队色优先使用材质槽 `team_color` 或 `TeamColor`。
- 选择圈、血条、阴影、可点击轮廓不能被模型替换破坏。
- 新资产进入前要明确 scale、offsetY、原点、朝向、包围盒和贴图预算。
- 新模型不能只在近景好看，必须在 RTS 镜头和缩放状态下可读。

## 建议更新方式

模型研究 session 可以追加新的 `ER-xxx`，但每条必须满足：

- 关联一个 runtime key 或明确的系统路径。
- 写清楚当前问题，不只写期望效果。
- 写清楚工程输入，主实现 session 能据此改代码或接资产。
- 写清楚玩家可观察的验收信号。

主实现 session 完成接入后，在状态列更新为 `Implemented` 或 `Verified`，并补一句验证方式。
