# 角色 09：技术美术 / 可读性负责人

> 用途：定义“在正式美术不足时，仍让画面可读、尺度合理、交战能看懂”的角色。  
> 当前状态：`部分覆盖`  
> 当前 owner：暂无稳定 owner

## 1. 角色使命

这个角色负责的是产品可读性，而不是“最终高保真美术”：

- silhouette
- 尺度
- 相机下识别度
- proxy / hybrid 方案
- 关键动画/效果可读

## 2. 阶段职责表

| 阶段 | 工作重点 | 主要交付 | 不负责 |
| --- | --- | --- | --- |
| `V0` | 规定资源格式和导入边界 | asset pipeline、占位资源规范、命名规则 | 不负责最终 art direction |
| `V1` | 支撑原型识别 | 基础单位/建筑/资源占位物、选中态对比 | 不负责复杂演出 |
| `V2` | 提升可信 slice 的视觉可读性 | 资源点、工人、建筑状态、基础攻击反馈可读 | 不负责所有 gameplay 判断 |
| `V3` | 让战场第一眼开始像 War3-like | 建筑尺度、树线、地表、单位轮廓、相机识别度 | 不负责地图布局本身 |
| `V4` | 支撑短局中的战斗和生产可视化 | 补兵、集结、交战、受击、塔防反馈 | 不负责 AI 行为设计 |
| `V5` | 支撑 roster 扩张后的识别 | 新单位分类、升级后外观区分、信息一致性 | 不负责 roster 内容定义 |
| `V6` | 支撑身份层表现 | 英雄辨识、法术层、物品/中立辨识 | 不负责能力创意 |
| `V7` | 把 proxy/hybrid 体系收成 Beta 可接受状态 | 素材替换优先级、视觉债务清单 | 不负责所有 bugfix |
| `V8` | 对外展示最低视觉门槛 | 截图、短视频、候选 build 的体面度 | 不负责发布渠道 |
| `V9` | 长期视觉迭代 | 风格统一、资产升级、技术美术优化 | 不负责产品路线 |

## 3. 当前阶段重点

当前项目最缺的不是“豪华美术”，而是：

1. 让 `V3` 的战场能被快速识别
2. 让建筑、树线、矿区、单位在当前镜头下读得清
3. 让占位资源体系服务真实判断，而不是遮盖问题

## 4. 交接与验收方式

- 输入：当前画面、相机截图、录像、资产限制
- 输出：可读性规则、替换优先级、proxy/hybrid 规范
- 核心指标：第一眼知道哪里是基地、哪里是资源、哪里在打架
