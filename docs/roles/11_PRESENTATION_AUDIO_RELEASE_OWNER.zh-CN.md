# 角色 11：演出 / 音频 / 外部包装负责人

> 用途：定义“从内部可玩走向外部可理解、可传播、可试听玩”的角色。  
> 当前状态：`后期需要`  
> 当前 owner：无稳定 owner

## 1. 角色使命

这个角色负责项目进入外部视野前的最后一层形态：

- 音频
- 演出
- README / 已知问题 / 分享材料
- 私测和公开候选包装

## 2. 阶段职责表

| 阶段 | 工作重点 | 主要交付 | 不负责 |
| --- | --- | --- | --- |
| `V0` | 不主导，仅给未来发布留接口 | 打包和素材留白建议 | 不负责基础工程本身 |
| `V1` | 不主导，仅记录原型演示所需最小包装 | 原型演示素材需求 | 不负责原型系统实现 |
| `V2` | 提前记录未来外部理解障碍 | 体验说明缺口、候选文案需求 | 不负责可信 slice 判断 |
| `V3` | 与可读性/UX 协同，准备最小演示表达 | 简短演示路径、截图要点 | 不负责地图/HUD 设计 |
| `V4` | 支撑短局 alpha 的外部观测 | playtest 说明、观察脚本、最小音频/反馈需求 | 不负责 AI 或系统实现 |
| `V5` | 开始规划 Beta 候选包装 | 教程、对外口径、展示重点 | 不负责内容范围决定 |
| `V6` | 让身份层有表达通道 | 英雄/法术/中立层的音频/演出规划 | 不负责能力设计 |
| `V7` | 做外部候选前的包装收口 | README、known issues、演示录像、基本音频层 | 不负责所有 bugfix |
| `V8` | 成为主角 | 私测包、公开 demo 包、FAQ、披露口径、反馈收集 | 不负责发布平台本身 |
| `V9` | 长期对外沟通和版本包装 | patch notes、对外变更说明、长期 presentation | 不负责总体 roadmap |

## 3. 当前阶段重点

当前不应该抢跑到完整 release 包装，但应该开始铺垫：

1. 未来外部试玩最少需要哪些说明
2. 当前最影响外部理解的 presentation 缺口
3. `V8` 需要提前准备哪些资产和文档

## 4. 交接与验收方式

- 输入：候选 build、已知问题、截图/录像、版本范围
- 输出：README、分享说明、已知问题披露、演示材料、音频/演出优先级
- 核心指标：外部人不需要内部语境，也能理解这是什么、怎么玩、哪里还不完整
