# 角色 07：AI / Match Loop 设计工程师

> 用途：定义“让 AI 和整局节奏成立”的角色。  
> 当前状态：`部分覆盖`  
> 当前 owner：GLM + Codex

## 1. 角色使命

这个角色负责两个层面：

1. AI 是否遵守和玩家一样的规则
2. 一局从 opening 到 pressure 到结束是否有弧线

## 2. 阶段职责表

| 阶段 | 工作重点 | 主要交付 | 不负责 |
| --- | --- | --- | --- |
| `V0` | 建立 AI 调试和验证入口 | AI regression、日志入口、可复现 harness | 不负责总体 infra 全部职责 |
| `V1` | 做出最小对手 | 基础采集、建造、训练、进攻脚本 | 不负责所有单位动作实现 |
| `V2` | 保证 AI 同规则可信 | 资源、建造、训练、攻击不靠作弊假装成立 | 不负责最终产品体验判断 |
| `V3` | 让 AI 理解新战场语法 | 开局布局、rally、矿工分配、出兵路径 | 不负责地图布局本身 |
| `V4` | 建立短局弧线 | opening、第一波压力、恢复、reinforce、收尾 | 不负责所有视觉反馈 |
| `V5` | 支撑战略骨架 | build order、扩张、tech、timing、counter 选择 | 不负责内容范围决策 |
| `V6` | 吸收身份层复杂度 | 英雄、法术、中立目标、物品逻辑 | 不负责能力设计本身 |
| `V7` | 做 beta 候选 AI 收口 | exploit 修复、行为稳定、难度基线 | 不负责外部运营 |
| `V8` | 对外测试对手稳定性 | 私测 demo 对手质量、失败案例收敛 | 不负责 release 包装 |
| `V9` | 长期对手演进 | 新策略、新模式、新难度层 | 不负责总体 roadmap |

## 3. 当前阶段重点

当前阶段 AI 最关键的任务不是“变强”，而是：

1. 继续对齐经济和战场合同
2. 让 AI 在 `V3` 的基地语法里不显得愚蠢
3. 为 `V4` 的短局弧线打底

## 4. 交接与验收方式

- 输入：战场语法、系统合同、录像、失败模式
- 输出：AI 行为、回归、日志、行为解释
- 核心指标：AI 不是脚本木偶，而是可信对手
