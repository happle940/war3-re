- `heroSkillPoints` 是可见数据字段，初始化为 1，每次升级增加 1。

- Holy Light 目前只有等级 1 的固定行为。
- 圣光等级 2/3、学习 UI、能力等级存储、光环、终极技能、其他圣骑士能力、其他英雄均未实现。

## 2. HERO11 合同边界

### 2.1 技能点消费机制

- 技能点消费必须通过玩家可见的 **命令卡 (command-card)** 或英雄面板入口显式触发。
- 每次消费 1 个技能点，增加 Paladin 拥有的一个能力等级 1 级。
- 消费前必须检查 `heroSkillPoints > 0`。
- 消费后 `heroSkillPoints -= 1`。

### 2.2 第一个实现目标：Holy Light

- HERO11 的第一个运行时目标是 **仅 Holy Light** 的等级提升。
- Holy Light 等级 2/3 的具体治疗效果数值由 `HERO11-SRC1` 来源边界确定。
- 等级提升后，施放 Holy Light 使用对应等级的治疗量。

### 2.3 复活持久性

- 已学能力等级 **必须** 在死亡和复活过程中保留。
- 复活后 Paladin 的 Holy Light 等级不变。

### 2.4 回归边界

以下已接受的行为 **不得** 退化：

| 系统 | 回归要求 |
|------|---------|
| HERO7 圣光合法性 | 技能学习不改变圣光施放条件（友方目标、射程、mana 消耗合法性） |
| HERO9 死亡状态 | 技能学习不改变 `isDead` 语义；死亡英雄不能学习技能 |
| HERO9 复活 | 复活费用/时间继续按 `heroLevel` 缩放，能力等级不独立影响复活 |
| HERO10 XP/升级 | 升级继续正确授予技能点 |
| HERO10 唯一性 | 仍只允许一个 Paladin |
