# 详细角色计划 10：内容 / 战略骨架设计师

> 角色摘要见：`docs/roles/10_CONTENT_STRATEGY_DESIGNER.zh-CN.md`  
> 详细计划对齐：`D6-D7` 主责，`D5` 次责

## 1. 研究范围

这个角色研究的是：

- 内容怎么长，才会长出策略，而不是长出杂音
- roster、tech、timing、counter 的最小骨架是什么
- War3-like 身份层应该何时引入

## 2. 大版本主责图

| 大版本 | 主责级别 | 关注点 | 主要输出 |
| --- | --- | --- | --- |
| `V0` | 低 | 未来内容约束 | scope notes |
| `V1` | 低 | 最小循环缺口记录 | missing-semantics note |
| `V2` | 中 | 当前 slice 的战略缺失 | strategy gap list |
| `V3` | 中 | 基地语法和兵种语义耦合 | content-space note |
| `V4` | 高 | 短局最小战略选择 | alpha strategic choices |
| `V5` | 极高 | 战略骨架扩张 | roster/tech/counter plan |
| `V6` | 极高 | 身份系统引入 | identity system scope |
| `V7` | 高 | content-complete 收口 | content freeze plan |
| `V8` | 中 | 对外展示内容范围 | demo scope |
| `V9` | 中 | 长期扩展 | content roadmap |

## 3. 里程碑详细计划

| 里程碑 | 该角色必须完成的动作 | 关键交付 |
| --- | --- | --- |
| `M0` | 记录未来扩张不要踩的结构边界 | content constraints |
| `M1` | 记录最小 RTS 缺失但不抢做 | missing semantics register |
| `M2` | 形成当前 strategic absence 清单 | post-M2 gap note |
| `M3` | 判断基地语法如何支持不同内容语义 | content-space mapping |
| `M4` | 定义短局需要的最小战略选择 | alpha strategy brief |
| `M5` | 主导 roster / tech / visual-content 方向选择 | direction packet |
| `M6` | 决定 demo 展示哪些内容、屏蔽哪些内容 | demo scope decision |
| `M7` | 确保 hardening 不封死未来内容路径 | future-content guardrails |

## 4. 当前到 M7 的具体动作

1. 先写清 `V5` 以前不该乱长的内容边界。
2. 在 `M5` 前准备三条路线的真实代价：单族精修、双族对打、系统沙盒。
3. 用“是否增加决策深度”作为内容准入标准。

## 5. 不能替别人吸收的事情

- 不能在 `G1-G3` 未稳时强行扩内容
- 不能把按钮变多等同于战略变深
- 不能自己决定 release 节奏
