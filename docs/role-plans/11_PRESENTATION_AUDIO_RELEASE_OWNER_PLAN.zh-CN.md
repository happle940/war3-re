# 详细角色计划 11：演出 / 音频 / 外部包装负责人

> 角色摘要见：`docs/roles/11_PRESENTATION_AUDIO_RELEASE_OWNER.zh-CN.md`  
> 详细计划对齐：`D8` 主责，`D4`、`D7` 次责

## 1. 研究范围

这个角色研究的是：

- 外部人怎么看懂这个项目
- 哪些体验缺口必须诚实披露
- 什么程度能私测，什么程度才配公开

## 2. 大版本主责图

| 大版本 | 主责级别 | 关注点 | 主要输出 |
| --- | --- | --- | --- |
| `V0` | 低 | 为未来包装预留最小接口 | packaging notes |
| `V1` | 低 | 原型演示说明 | demo notes |
| `V2` | 中 | 记录未来解释债 | explanation debt register |
| `V3` | 中 | 建立最小截图/录像/讲解路径 | capture path |
| `V4` | 高 | 支撑 alpha playtest 的表达层 | playtest framing |
| `V5` | 高 | 准备身份和方向的外部表达 | direction-facing packet |
| `V6` | 高 | 支撑身份层展示和包装 | presentation plan |
| `V7` | 高 | 候选版文档和说明收口 | candidate docs |
| `V8` | 极高 | private/public share packaging | release packet |
| `V9` | 中 | patch notes 和长期外部表达 | release comms roadmap |

## 3. 里程碑详细计划

| 里程碑 | 该角色必须完成的动作 | 关键交付 |
| --- | --- | --- |
| `M0` | 记录未来发布所需最小结构 | packaging note |
| `M1` | 给第一可玩切片准备最小演示说明 | M1 demo framing |
| `M2` | 记录仍无法对外解释的体验缺口 | explanation debt register |
| `M3` | 为 War3-like 战场切片准备 capture/review 方式 | G2 presentation path |
| `M4` | 为人机 Alpha 准备试玩说明和观测脚本 | alpha playtest packet |
| `M5` | 参与视觉/内容方向的外部表述 | direction-facing brief |
| `M6` | 主导 private/public 候选包装 | release packet |
| `M7` | 更新 hardening 后的已知问题与描述边界 | updated disclosure set |

## 4. 当前到 M7 的具体动作

1. 不抢跑做 marketing，而是先把未来 `M6` 的披露口径定义清楚。
2. 在 `M4` 前准备好可执行的试玩说明和 session log。
3. 在 `M6` 前保证 README 和 Known Issues 绝不夸大当前状态。

## 5. 不能替别人吸收的事情

- 不能把包装当产品完成度
- 不能替 QA 做 smoke 结论
- 不能替产品负责人决定公开分享等级
