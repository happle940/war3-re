## 2. 死亡状态运行时语义

### 2.1 英雄死亡触发

当英雄单位 HP 降至 0 时：

1. **停止行动**：清除攻击目标、移动目标、施法目标，进入死亡状态
2. **停止被选为攻击目标**：敌方单位不再将死亡英雄视为有效攻击目标
3. **保持单位存在**：死亡英雄不从 `units` 数组中移除
4. **设置 `isDead = true`**：通过运行时赋值而非数据初始化
5. **HP 保持 0**：不触发移除逻辑
6. **保持唯一性**：死亡英雄仍计入该英雄类型的唯一性检查，防止召唤第二个同类型英雄

### 2.2 死亡英雄的运行时表现

| 属性 | 死亡状态 |
|------|---------|
| `hp` | 0 |
| `isDead` | `true` |
| 可被选中 | 否（或可选：可选中但无法操作） |
| 可被攻击 | 否 |
| 可施法 | 否 |
| 可移动 | 否 |
| 占用英雄名额 | 是 |
| 占用人口 | 是 |
| 命令卡 | 空或死亡状态显示 |

### 2.3 唯一性与死亡的关系

当前 HERO6B 唯一性检查：

```typescript
const hasAliveHero = this.units.some(
  u => u.team === 0 && u.type === heroKey && !u.isBuilding && u.hp > 0,
)
```

此检查使用 `hp > 0`，意味着死亡英雄（`hp === 0`）不触发唯一性阻止，理论上可以召唤第二个。

**必须修改**：召唤唯一性检查不能继续使用 `hp > 0`，也不能简单改成 `!u.isDead`。正确合同是把“召唤”和“复活”拆成两种判断：

- 新召唤检查：同队伍同类型英雄记录只要存在（不论 `hp` 或 `isDead`），就阻止新召唤。
- 复活检查：同队伍同类型英雄记录存在且 `isDead === true`，才允许祭坛显示复活入口。
- 同类型英雄存在但已死亡 → 允许复活（不是新召唤），仍阻止新召唤。

推荐伪代码：

```typescript
const hasExistingHero = this.units.some(
  u => u.team === team && u.type === heroKey && !u.isBuilding,
)
const deadHero = this.units.find(
  u => u.team === team && u.type === heroKey && !u.isBuilding && u.isDead === true,
)
```

## 3. 祭坛复活机制

### 3.1 复活前提条件

- 玩家拥有已完成的祭坛
- 该玩家有对应类型的死亡英雄
- 复活不在队列中（无重复排队）

### 3.2 复活按钮

完成祭坛的命令卡应显示：
- 若无死亡英雄：显示英雄召唤按钮（当前行为不变）
- 若有死亡英雄：显示复活按钮（替代或补充召唤按钮）

### 3.3 复活规则（需源边界确认的项标记为 `[SRC]`）

| 规则 | 当前状态 | 说明 |
|------|---------|------|
| 复活费用 | `[SRC]` 需源边界确认 | 示例占位值：英雄造价 × 50%，非生产值 |
| 复活时间 | `[SRC]` 需源边界确认 | 示例占位值：与训练时间相同，非生产值 |
| 人口要求 | 复活不额外消耗人口 | 英雄已占用人口 |
| 队列行为 | 使用训练队列 | 复活排入 `trainingQueue`，与召唤共享队列 |
| 复活产出 | 恢复现有死亡英雄 | HP 恢复，`isDead = false`，不创建新单位 |

### 3.4 复活后的英雄状态

| 属性 | 复活后值 |
|------|---------|
| `hp` | 恢复到一定比例 `[SRC]`（占位：maxHp × 50%） |
| `isDead` | `false` |
| `mana` | 保持死亡时的值或恢复到一定比例 `[SRC]` |
| `heroLevel` | 不变 |
| `heroXP` | 不变 |
| `heroSkillPoints` | 不变 |
| 位置 | 祭坛附近出生点 |

## 4. 源边界要求

本合同**不定义**最终的复活费用和时间。以下项目必须在 HERO9-SRC1 源边界任务中确认：

| 项目 | 状态 | 说明 |
|------|------|------|
| 复活费用公式 | `source-boundary required` | 可能与英雄等级相关 |
| 复活时间 | `source-boundary required` | 可能与英雄等级相关 |
| 复活后 HP 比例 | `source-boundary required` | ROC 原版值需确认 |
| 复活后 mana 比例 | `source-boundary required` | ROC 原版值需确认 |
| 死亡英雄是否可见/可选中 | `source-boundary required` | War3 中死亡英雄留下尸体/光环 |

本合同中的任何具体数值（如"50%"）仅为说明性占位，**不可**作为生产值使用。

## 6. 禁区

本合同**不打开**以下范围：

- 英雄经验/升级系统
- 技能点消耗/技能选择
- 其他三个英雄（大法师、山丘之王、血法师）
- 酒馆（Tavern）系统
- 物品栏/物品系统/商店
- 光环系统
- 圣光术自动施放
- AI 英雄策略
- 英雄专属视觉/音效/资源
- 空军单位
- 第二阵营
