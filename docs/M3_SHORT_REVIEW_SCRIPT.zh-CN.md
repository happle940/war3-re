# M3 Short Review Script：空间语法、镜头与可读性

> 用途：给用户快速判断 `M3`。不用读内部文档，不用完整打一局，只看当前画面是否已经能支撑后续 Alpha 对局判断。

## 1. 评审目标

请判断一件事：

**它第一眼是否开始像一个 War3-like RTS 战场，而不是网页原型或随机调试场景。**

M3 只看空间语法、默认镜头和可读性。不要在这里评最终美术、AI 强度、完整胜负或内容量。

## 2. 按顺序看这 7 项

1. **第一眼**
   - 打开后先停 5 秒，不操作。
   - 看它是否马上读成一个 RTS 基地 / 战场，而不是网页 demo。

2. **基地核心**
   - 找 Town Hall、worker、Farm、Barracks。
   - 看 Town Hall 是否是清楚的基地中心，周围建筑和单位是否有层级。

3. **经济路径**
   - 看 Town Hall 和金矿之间的距离。
   - 应该能读出采集路线；不能远到断开，也不能挤成一团。

4. **树线和出口**
   - 看树线、通道、基地出口是否让地图有方向感。
   - 不要求地形精美，但不能像随便摆放。

5. **比例层级**
   - 对比 worker、footman、Farm、Barracks、Town Hall。
   - 小单位、普通建筑、核心建筑的体量关系要基本成立。

6. **默认镜头和 HUD**
   - 默认视角应能看到基地核心、矿区和主要操作区。
   - HUD 不应挡住关键对象，也不应让主战场读不清。

7. **反馈和视觉一致性**
   - 点选单位 / 建筑，看看选择圈、血条、命令反馈是否可读。
   - 即便还是 proxy，也应像同一套临时视觉，而不是混搭素材。

## 3. 明显失败条件

出现任一项，就可以判 `M3` 失败：

- 第一眼不像 RTS 战场，更像网页原型、测试页或随机摆物。
- Town Hall、金矿、worker、树线或主要操作区在默认镜头下读不清。
- 基地、金矿、树线、出口关系没有空间语法，像随机摆放。
- 单位和建筑比例明显错，影响判断谁是工人、兵、普通建筑、核心建筑。
- HUD 或命令卡遮挡主战场关键对象。
- 选择圈、血条、命令反馈缺失、错位或比例不可信。
- proxy 视觉混乱到无法判断项目是否值得继续朝 War3-like 方向推进。

## 4. 怎么给结论

请只选一个：

- `通过`：第一眼像 RTS 战场；基地 / 金矿 / 树线 / 出口有明确空间语法；比例、镜头、HUD、反馈足够支撑进入 M4。
- `通过但有美术债`：空间和可读性成立，但 proxy 粗糙、细节不统一、部分比例或反馈需要后续打磨。
- `失败：比例问题`：单位和建筑体量层级明显不成立。
- `失败：地形问题`：基地、金矿、树线、出口关系读不出来。
- `失败：镜头问题`：默认 framing 或 HUD 让关键对象看不清。
- `失败：视觉身份问题`：proxy 视觉太混乱，无法判断 War3-like 方向是否成立。

如果选失败，请附一句最短原因，例如：“默认镜头看不到矿区”或“Town Hall 和 Farm 体量太接近”。

## 5. M3 先不用担心什么

- 不评价最终美术质量。
- 不要求正式资产已经接入。
- 不评价 AI 强不强。
- 不要求完整胜负闭环。
- 不要求内容量足够。
- 不评价公开 demo 是否可发。

这些分别属于 M4、M5、M6 或后续 polish。M3 只判断当前画面是否已经有足够清楚的 RTS 空间语法和可读性。
