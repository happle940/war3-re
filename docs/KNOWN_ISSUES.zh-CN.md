# Known Issues

> 用途：给后续私测 / 外部试玩 / release packet 一个稳定的已知问题来源。
> 原则：只记录当前真实存在、会影响体验理解或反馈质量的问题；不把未来愿望写成 issue。

---

## 当前阶段定位

- 当前版本是 `War3-like browser RTS private alpha`，不是公开发行版。
- 当前顶层路线以 `PLAN.zh-CN.md` 的 `R0-R15` Roadmap 为准；旧 V8/V9 证据可以作为历史实现来源，但不再作为新任务入口。
- 当前 R1-R15 都已有私有 alpha 第一轮运行时闭环。当前问题集中在深度体验：开局引导质量、RTS 操控细节、Human T2/T3 深度、War3-like 身份细节、真实动画/音频表现、外部试玩深化；AI 长局可信度保留为后续项，当前不抢主线。
- R15 诊断反馈包已接入 War3 全局差距雷达；这能帮助每轮汇报差距，但不代表这些差距已经关闭。
- 线上 GitHub Pages 版本适合 milestone 回看、受控试玩和反馈收集，不适合当作“完整 Warcraft-like demo”对外扩散。

---

## 会影响外部试玩理解的问题

### 1. 产品旅程已闭合，但还不是发布级包装

- 当前已有 menu、briefing、pause、results、return/reload 等基础壳层。
- 主菜单、暂停和结果页已有试玩信息入口，能看到版本边界、Known Issues 和诊断反馈包。
- 但它还没有达到 War3.exe 式的完整游戏流程。
- 设置、帮助、局前配置、发布级恢复文案、真实浏览器矩阵、可执行性能阈值、反馈运营和战报深度仍不足。

### 2. 开局体验已有闭环，但仍需要产品化

- 当前默认局可以开始，基地、金矿、树线、worker、HUD、命令卡和 R1-R6 基础快照都有 runtime proof。
- 但第一屏和第一分钟仍偏工程 alpha：主菜单氛围、提示、镜头、战场层次、下一步行动引导都需要继续打磨。

### 3. 完整 Human 与数值系统仍未完成

- 当前已经有不少 Human-like 单位、建筑、科技、英雄和能力进入运行时。
- 这些能力已经整理成当前玩家可理解的人族路线，并补出 T1/T2/T3 解锁、下一步阻断原因、路线 icon 和解锁完成 cue，能形成私有 alpha 短局。
- 但它还不是完整 Human race，也还没有证明稳定 10-15 分钟的完整数值体验。

### 4. 视觉资产和动作仍不是最终表现

- 当前版本重心是系统合同、可玩性和合法来源，不是最终美术。
- 外部试玩者看到的建筑/单位视觉，只能代表当前可读性与方向，不代表最终资产品质。
- 当前已有 clip-aware runtime、procedural 最小单位动作语言、玩家感知反馈快照、真实动作/音效素材门禁和当前合同全覆盖基础 GLB clips / wav cues，但这仍不是最终表现；建筑施工/升级、采集细节、攻击/施法/死亡的动作质量、音频 identity 和发布级资产批准仍需要补齐。

### 5. War3 全局差距已可观测，但不能被误读为已追平

- 当前 R15 反馈包会输出 `War3 gap radar`，覆盖前门、第一分钟、操控、经济、战斗、Human、英雄、地图/Fog/物品、AI、视觉音频、试玩发布和架构。
- 它的作用是暴露缺口和排序，不是证明完整 War3-like demo。
- AI 长局可按当前优先级暂缓；R14 已能观测命令、战斗、技能、死亡、建造、cue 覆盖和真实素材门禁，并已有当前合同全覆盖 asset-backed clips/cues，但视觉质量、动作质量、音频 identity、Human T2/T3、命令失败原因、完整 Fog 和发布级包装仍会作为缺口保留。

### 6. 体验解释仍部分依赖项目上下文

- 核心操作和试玩反馈入口已经可用，但试玩者如果没有最小背景说明，仍可能误解当前范围：
  - 这是系统优先的 alpha
  - 不是完整内容版
  - 不是最终视觉版
  - 后续路线以 R0-R15 Roadmap 推进，不代表所有 Roadmap 阶段已经完成

### 7. 代码结构仍需要渐进重构

- `Game.ts` 仍承担过多运行时职责，继续堆功能会增加回归风险。
- 后续重构以 `R-ARCH` 横切主线推进，绑定具体 Roadmap 切片。
- 不应把这理解成要暂停产品体验开发去做一次性大重写。

## 已知的工程 / 发布层限制

### 1. Release packet 不是 public release

- README、PLAN 和 Command Center 已开始对齐到 R0-R15 Roadmap。
- 但当前状态仍是受控试玩和 Roadmap 驱动开发，不是公开发布。
- 不能写成完整 War3、完整 Human race、public demo released 或 release-ready。

### 2. 历史 smoke path 仍是回归参考

- `tests/v8-demo-path-smoke.spec.ts` 覆盖入口、开始、暂停返回/重开、结果返回和范围说明。
- `tests/v8-release-candidate-stability.spec.ts` 覆盖 V7 内容数据、训练/战斗、普通入口会话、HUD/命令状态和清理恢复。
- `tests/v9-baseline-replay-smoke.spec.ts` 可作为历史 baseline replay 参考。
- 新任务仍应从 R0-R15 Roadmap 切片，再选择对应 focused runtime 测试。

### 3. Roadmap 已建立，但未完成

- 当前固定顶层方向是 `R0-R15`。
- 结构化重构方向是 `R-ARCH`，服务于 `R0-R15`，不是独立取代玩家体验路线。
- 后续仍要逐项证明功能、用户体验、AI、HUD、数值、产品壳层和外部试玩能力。
- 第二阵营、多人和公开发布仍不进入近期优先级。

## 不应再作为“已知问题”重复报告的事项

下面这些问题已经被当前 runtime 合同覆盖，不应再把它们当成未处理状态：

- 单矿最多 5 个有效采金工人，第 6 个进入等待/递减收益
- crowded goldmine 左键可选中、右键可采集
- clear rally 不再制造假 rally
- 建造中断后可续建，取消建造可释放占地并清理 builder 状态
- AI 金矿饱和策略已按 5 工人合同工作
- AI impossible placement spam 已被修复并通过全量 runtime

## 当前更适合收集的反馈类型

- 打开网页第一屏是否像游戏，而不是工程说明页。
- 第一局能否自然开始，第一分钟是否知道下一步该做什么；如果知道，哪里还像工程 alpha。
- 命令、采集、建造、生产、战斗哪里最破坏信任。
- Human 单位、建筑、科技、英雄和 AI 是否能形成 10-15 分钟短局。
- 返回、重开、结果、关闭网页前后的状态是否清楚。
- 试玩信息里的诊断反馈包是否足够描述版本、浏览器、地图、结果和阻断点。

## 更新规则

- 每次准备私测 / 对外试玩前都要重读并更新本文件。
- 如果某个问题已经被 runtime 合同锁定并在当前工作树验证通过，应从“当前 issue”转为“已覆盖说明”。
- 如果某个问题需要人的主观判断，必须明确写它还缺的是 `human gate`，而不是假装成工程未完成。
