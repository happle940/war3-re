## M1 第一个可玩 RTS 切片

状态：已完成。

你的结论：`带视觉债通过`。

这代表：游戏现在可以作为一个小 RTS 玩 5-10 分钟，但视觉和 War3 系统完整度仍有债务。

你确认了：

- 默认视角下，单位和核心建筑基本可见。
- 选择、移动、停止、攻击移动、建造基本听话。
- 能采集、建造、训练、打架。
- AI 有压力。
- 基地布局基本可玩。

你同时提出的新问题：

- 兵营建造到一半会停，没法续建。
- 箭塔没有攻击力。
- 单位没有碰撞体积。
- 人口卡住时，人口建筑/命令反馈不清楚。
- 没有建造取消。
- 更高层的问题是：还没有系统性向 War3 规则对齐。

这些问题不作为 M1 失败处理，而是升级为 M2 的核心任务。

## M2 War3 核心系统对齐

状态：下一个大里程碑。

### 这个里程碑要回答的问题

它是否开始遵守一个 Warcraft III 玩家预期中的 RTS 规则，而不是只有零散命令？

你这次反馈暴露的核心不是单个 bug，而是系统层缺口：

- 建造有了，但建造生命周期需要完整契约。
- 箭塔有了，但静态防御战斗系统需要完整契约。
- 人口有了，但命令卡阻塞原因需要清楚表达。
- 移动有了，但单位物理存在感需要基础契约。
- 放置有了，但取消和退款规则需要完整契约。
- 战斗有了，但手动移动、停止、hold、attack-move 不能被自动索敌立即抢回。

### 到 M2 前，我们必须先自己完成

Codex/GLM 先确认：

- 建筑建到一半停了，可以由有效农民继续建。
- 建造可以取消，释放占地，释放建造者状态，并按明确规则退款。
- 箭塔有射程、伤害、冷却和自动索敌。
- 人口/资源不足时，命令卡能解释为什么不能点。
- 单位不会长期堆在同一个点上，有基础碰撞/分离。
- 战斗中的单位能服从玩家移动、停止、hold、attack-move 指令。
- 这些规则都有 runtime 测试保护。

### M2 时你确认什么

你只需要判断：

- 中断建造后恢复是否自然。
- 建造取消是否符合直觉。
- 箭塔是不是像防御建筑，而不是摆设。
- 点不了的命令是不是说得清原因。
- 单位是不是有一点实体感。
- 打架中的单位是不是仍然听你的指令。

你只需要选一个：

- `通过`
- `带视觉债通过`
- `失败：建造系统`
- `失败：战斗系统`
- `失败：命令卡反馈`
- `失败：碰撞/体积`
- `失败：战斗控制`

状态：计划中。

这个项目后续到底应该长成什么样？

### 到 M5 前，我们必须先自己完成

Codex/GLM 先确认：

- 当前 gameplay loop 值得继续扩展。
- 合法素材来源和许可风险已经整理。

### M5 时你确认什么

你只需要选方向：

视觉方向：

- `proxy-first`：先保可读性，用自制/程序化 proxy。
- `asset-pack-first`：优先接合法资产包，接受短期风格不统一。
- `hybrid`：现在用 proxy 保 gameplay，后续逐步替换合法资产。

内容方向：

- `一个打磨好的人族切片`
- `人族 vs 兽族`
- `系统优先沙盒`

产品目标：

- `继续打磨手感`
- `扩内容`
- `准备公开 demo`

默认建议：`hybrid + 一个打磨好的人族切片 + 继续打磨手感`。
