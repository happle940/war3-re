# M4 Alpha Playtest Script：人机 Alpha 短局

> 用途：给用户快速判断 `M4`。目标是一局短的人机 Alpha，判断 match loop 是否成立，不要求完整内容体验。

## 1. 试玩目标

请判断一件事：

**这局是否有开始、中期压力、可理解的结束或卡点，并且玩家没有被控制问题或 AI 低级坏死阻断判断。**

M4 不评价最终视觉、不评价完整内容量、不要求公开 demo 品质。它只看人机 Alpha 对局是否已经值得继续推进。

## 2. 单局试玩步骤

1. **开局停 5 秒**
   - 看基地、工人、资源点和敌我关系是否能理解。
   - 如果一开始就不知道自己该做什么，记录为可读性 / 引导问题。

2. **做经济开局**
   - 选 worker，右键金矿或树木。
   - 看采集是否启动，资源是否开始变化，HUD 是否能解释当前状态。

3. **补一点生产**
   - 用 Town Hall 训练 worker，或用 Barracks 训练 footman。
   - 看资源扣除、队列、单位出生是否正常。

4. **尝试基本防守 / 出兵**
   - 训练 2-3 个可战斗单位，或准备防御建筑。
   - 观察自己是否有时间和方式应对 AI，而不是一开始就被系统锁死。

5. **接触 AI 压力**
   - 等 AI 出兵或主动向敌方方向推进。
   - 看 AI 是否会采集、出兵、推进或恢复，而不是静止样机。

6. **测试控制权**
   - 战斗中右键拉走单位，再按 S 停止，必要时试 H 驻守。
   - 看单位是否听指令，auto-aggro 是否过快抢回控制。

7. **继续到一个结果或卡点**
   - 玩到你赢、输、明显僵持，或遇到无法继续的问题。
   - 重点不是一定要打完，而是你能否说清为什么会这样。

## 3. 重点观察

### 控制

- 选择、右键、命令队列、建造、训练、集结是否可靠。
- 战斗中单位是否还能移动、停止、驻守。
- 控制问题是否让对局显得不公平，或让玩家无法防守 / 补兵 / 反打。

### AI 压力

- AI 是否会采集、维持经济、出兵或推进。
- AI 第一波后是否永久卡死。
- AI 损失工人或建筑后，是否还能恢复基本经济和出兵。
- AI 的失败方式是否低级到让你无法评价这局。

### 结束清晰度

- 赢、输、僵持或卡住以后，你能不能说清原因。
- 结束状态是否有明确表达，还是只能靠猜。
- 如果没有明确结束，至少卡点是否可复现、可解释。

## 4. 怎么报告

请用下面格式回报：

```text
M4 结论：通过 / 带债通过 / 失败：控制问题 / 失败：AI 问题 / 失败：节奏问题 / 失败：胜负不清晰
一句话原因：
最影响判断的 1-3 个问题：
是否能继续进入后续里程碑：能 / 不能 / 能但带债
```

判定口径：

- `通过`：有开始、中期压力、可理解结果；控制没有明显不公平；AI 没有低级坏死。
- `带债通过`：核心 loop 成立，但节奏、反馈、AI 压力或胜负表达还有不阻断判断的债。
- `失败：控制问题`：玩家因为控制、HUD、队列、建造或移动反馈无法公平操作。
- `失败：AI 问题`：AI 停摆、恢复失败、经济或出兵坏死。
- `失败：节奏问题`：有功能但没有一局感；没有中期压力、无限拖延或不知道该推进什么。
- `失败：胜负不清晰`：赢、输或卡住都无法理解原因。

## 5. 立即踢回工程合同的问题

遇到这些问题，不要当成 taste 争论，直接记录复现并回工程合同：

- 选中失败、右键命令不生效、命令队列丢失。
- 资源、供给、训练成本、单位出生、采集循环明显错误。
- 建造取消 / 中断 / 续建导致资源、占地或 builder 状态卡住。
- AI 工人损失后不恢复、建筑损失后不重建、第一波后永久停摆。
- 单位死亡后引用未清理、路径 / 占地导致不可恢复卡死。
- HUD 状态错误、命令卡误导、胜负状态缺失或与实际不一致。
- 同一操作在相同初始状态下结果不稳定。

这些问题只要能复现，就应写成 contract / regression task，再回到 M4 判断。
