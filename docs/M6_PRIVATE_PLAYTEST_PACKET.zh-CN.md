# M6 Private Playtest Packet：少量私测说明

> 状态：私测候选包。本文只用于判断和组织少量私测，不代表 `M6` 已批准公开分享。

## 1. 现阶段私测适合谁

当前版本适合发给少数可信 tester，不适合公开扩散。

适合的人：

- 能接受这是 `gameplay alpha`，不是完整 demo。
- 愿意按指定路径试玩 5-10 分钟，而不是自由评价整个项目。
- 能区分系统问题、视觉 proxy、内容缺口和产品方向问题。
- 能给出具体复现、截图、浏览器环境或一句清楚的判断。
- 不会把链接当作正式 Warcraft-like 游戏传播。

不适合的人：

- 只会评价最终美术、内容量或“像不像成品”。
- 不愿意读最小上下文。
- 看到 proxy / fallback 视觉就会把反馈集中在美术质量上。
- 会把私测链接转发给没有上下文的人。

## 2. Tester 打开前必须收到的上下文

发送链接前，至少要告诉 tester：

- 这是 Warcraft-like RTS 的 `gameplay alpha`，目标是验证基础操控、经济、生产、AI 压力和可读性。
- 这不是完整发行版，不是公开 demo，也不是最终视觉版本。
- 当前视觉仍以 proxy / fallback 为主，只代表可读性和方向，不代表最终资产品质。
- 当前可能还没有完整、稳定、明确的 `victory / defeat` 结束表达。
- `M3` 的空间 / 镜头 / 视觉方向、`M4` 的完整人机 Alpha、`M5` 的产品方向、`M6` 的 release 判断都还需要人工确认。
- 试玩重点是：能不能打开、看懂开局、完成最小经济和生产闭环、感受到 AI 不是静止样机。

可直接发送的短说明：

```text
这是一个 RTS gameplay alpha 私测链接，不是公开 demo。
请只试玩 5-10 分钟，重点看：开局是否看得懂、worker 能否采集、Town Hall/Barracks 能否生产、AI 是否会动、HUD/控制是否误导。
画面仍是 proxy/fallback，不代表最终美术。胜负闭环可能还没完整表达。
请反馈具体问题和浏览器环境，不用评价完整内容量或最终美术。
```

## 3. 现在最有用的反馈

请优先收集这些反馈：

- 是否能正常打开：无白屏、无启动崩溃、初始 HUD 出现。
- 开局是否看得懂：默认镜头下能否分辨 Town Hall、worker、金矿、树线。
- 最小操作闭环是否成立：选中 worker，右键金矿开始采集，右键地面有移动反馈。
- 最小生产闭环是否成立：Town Hall 训练 worker 或 Barracks 训练 footman，资源扣除和单位出生正常。
- AI 是否像活着的对手：能采集、继续出兵或维持经济，不是空地图摆设。
- HUD / 点击 / 镜头是否明显误导：挡住关键对象、点不中核心对象、命令卡不可读。
- 哪个已知限制最影响理解：没有胜负表达、proxy 视觉、缺少上下文、视觉/镜头尚未人工确认。

反馈格式建议：

```text
浏览器 / 设备：
能否打开：能 / 不能
能否完成采集和生产：能 / 不能
AI 是否行动：是 / 否 / 不确定
最影响理解的 1-3 个问题：
是否适合继续私测：是 / 否 / 修完再说
```

## 4. 必须提前披露的已知限制

私测前必须透明说明：

- 当前仍是 `gameplay alpha`，不是公开发行版。
- 线上版本适合 milestone 回看和定向试玩，不适合当作完整 demo 对外扩散。
- 还没有完整、稳定、明确的胜负闭环表达。
- `M3` 的视觉 / 镜头结论还没有人工确认。
- 视觉资产仍以 proxy / fallback 为主。
- 体验解释仍依赖项目上下文；tester 如果不读说明，可能误解当前范围。
- release packet、README 外部入口、known issues 和 smoke path 仍是 M6 准备材料的一部分。
- 工程硬化仍在进行，当前版本不代表结构风险已经全部出清。

同时也要说明：已由 runtime 合同覆盖的问题，不应重复当成当前 issue 报告，例如采金饱和、crowded goldmine 点击、clear rally、建造续建 / 取消、AI 金矿饱和策略等。

## 5. 即使是私测也要阻断的问题

出现任一项，不应发送给私测 tester：

- 页面无法打开、白屏、启动崩溃，或需要本地手工步骤才能运行。
- 初始 HUD 不出现，或默认镜头下核心对象完全不可见。
- worker 不能被选中，或右键移动 / 采集没有可见反馈。
- Town Hall / Barracks 命令卡不可读，或最小训练闭环无法完成。
- 资源扣除、单位出生、采集循环存在明显破坏性错误。
- AI 完全不行动，tester 只能在空场景里操作。
- 存在不可恢复卡死、持续刷错、严重性能问题。
- HUD、点击、镜头问题一眼就污染反馈，例如点不中核心对象或 HUD 压住主视图。
- [KNOWN_ISSUES](./KNOWN_ISSUES.zh-CN.md) 没有更新。
- 私测说明没有交代 alpha 范围、proxy 视觉、已知限制和反馈边界。
- 用户尚未批准任何外部分享，而文档或消息把它描述成公开 demo。

## 6. 私测结论怎么记录

每轮少量私测结束后，只记录这三类结论：

- `可以继续私测`：smoke 通过，反馈集中在有价值边界内，没有 red line。
- `修完再私测`：核心路径大体可用，但有 1-3 个会污染反馈的问题。
- `停止外发`：存在启动、核心交互、生产闭环、AI 存活性或反馈污染 red line。

不要把“私测可以继续”写成“M6 已批准公开”。公开分享仍需要单独 release 判断。
