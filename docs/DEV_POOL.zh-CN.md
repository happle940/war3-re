# War3 RE 协作开发池

这份文档用于两个 Codex session 共同推进同一个 War3-like 浏览器 RTS 项目：主实现 session 继续做运行时和玩法，模型研究 session 聚焦模型、建模、资产可读性和导入约束。

## 边界

- 这里是开发池和共享事实，不是任务调度系统。
- 不恢复 GLM、双泳道、watch、看板、queue refill、task synthesis。
- 不把文档声称当成运行时事实。新增需求要能落到代码路径、资产路径、测试路径或试玩验收信号。
- 模型研究 session 默认不改运行时代码；如需改代码，先把建议写到本文或 `docs/ENGINEERING_REQUIREMENTS_POOL.zh-CN.md`。

## 当前可玩基线

- 网页可打开进入战局，基础 session/re-entry 已经不是纯空壳。
- R1-R6 基础体验已有当前闭环：前门、开局、第一分钟、操控、经济/生产和战斗底盘进入 `FoundationMilestoneSystem`、HUD 和 R15 反馈包。
- Human 经济可跑：农民采金、伐木、建造、回收资源、人口和训练链路可用。
- Human 基础建筑和单位已进入 runtime：`townhall`、`barracks`、`farm`、`tower`、`blacksmith`、`lumber_mill`、`workshop`、`arcane_sanctum`、`altar_of_kings`、`arcane_vault`；单位含 `worker`、`footman`、`rifleman`、`mortar_team`、`priest`、`sorceress`、`knight`、`militia`。
- 英雄线已具备 Paladin、Archmage、Mountain King 的可用技能入口、XP/等级、技能学习、死亡记录/复活底座、三英雄技能反馈、技能可用性矩阵、目标合法性提示、施法距离/范围/光标语义、主动目标合法/非法判定和复活尸体可读层，并已进入 R8 运行时快照。
- 中立营地、击杀经验、掉落物、英雄背包、Arcane Vault 购买药水/装备已经有 runtime proof。
- 简单 AI、训练、战斗、商店购买已经覆盖到五分钟级 integrated smoke。
- AI 已有短局压力快照：阶段、波次、下波倒计时、练级意图、商店购买和峰值压力会进入 HUD 与结果摘要。

- Human 路线整理已进入运行时：经济、兵营、英雄、支援、科技、后期六段路线会在 HUD 面板和快照里表达，并补出 T1/T2/T3 解锁、技术节奏阶段、战术角色、混编覆盖、兵种克制读法、科技收益、生产线概览、下一步动作和 power spike；命令卡按钮同步路线 tier、角色、focus，字母热键已可执行并能在禁用时反馈缺口原因，训练/建造/研究/商店/背包/技能/基础命令也已有稳定 `iconKey`、程序化图标、HUD 背包槽、命令状态徽标、冷却/生效 meter、法力缺口 badge 和目标模式高亮基线，不再只是一堆散文字按钮。
- R10 短局闭环已经由 runtime proof 验收：开局经济、生产、部队、英雄、地图目标、商店、AI 压力、决战、胜负摘要和重开重置能走完一局。
- 战场目标雷达已有 runtime 基线：我方基地、金矿线、树线、野怪营地、商店、敌方基地会进入 HUD，关键目标同步到小地图圈，并在世界视角显示目标 beacon。

- Fog / 侦察 / War3 身份已有 runtime 基线：可见/已探索区域、中立营地、掉落物、Arcane Vault 消耗品、回城卷轴和目标侦察会进入同一身份快照。
- 产品壳层已有当前闭环：主菜单、设置、briefing、暂停、结果、返回、重开、偏好保存、关闭保护和上局摘要都有明确状态语义。
- 视觉 / 音频身份已有 R14 当前闭环：低模资产基线、War3-like HUD 皮肤、技能视觉反馈、施法预览语言、主动目标合法性反馈、复活尸体 marker/范围环、技能特效爆发、结果页视觉复盘、目标/压力/技能 cue 总线、选中/命令/命中/血条/状态表现层、玩家感知反馈层、最小单位动作语言、clip/fallback 动作来源、真实动作/音效素材门禁和当前合同全覆盖 asset-backed clips/cues 会进入同一身份快照。
- 外部试玩准备已有 R15 当前闭环：主菜单、暂停和结果页都能打开试玩信息，版本边界、已知问题、诊断反馈包、性能预算信号、兼容矩阵、错误缓冲、反馈分类、恢复按钮、Human 解锁层、战场表现层和关闭保护进入同一快照。
- War3 全局差距雷达已有运行时入口：`War3GapSystem` 把 R1-R15 证据汇成 12 个玩家旅程差距域，并进入 R15 诊断反馈包，用于每轮追赶复盘。

这些能力仍然不是 War3 等价物：它们证明“浏览器里能玩起来”，但还没证明“玩家能像打开 war.exe 一样自然理解、信任和复盘一局”。

## 当前不足的共享判断

- 模型和视觉身份已有低模基线、cue 总线和当前合同全覆盖基础 clips/cues，但仍不是最终美术：很多资产仍是工程 baseline，远距离剪影、动作质量和真实音效 identity 不稳定。
- 建筑科技树仍不完整：`keep`、`castle` 已有模型、运行时入口和节奏/角色反馈，但仍更接近二本/三本基线，不应被当作完整 T2/T3 体验。
- 动画不足：当前已有 clip-aware runtime、procedural 最小动作语言、玩家感知反馈快照和素材门禁，且 12 个核心单位已具备基础 transform clips；但建筑施工/升级、采集细节、技能前摇/后摇和发布级骨骼动作仍需要更高质量资产。
- 地图目标和侦察已有基线：出生点、矿线、树线、中立营地、商店、敌基、Fog 可见/已探索都有形，但还没形成标准 melee 地图的探索节奏、路径压力和高低差遮挡。
- UX 信任不足：选择、命令、施法、购买、失败原因、冷却、范围、目标合法性、Human 混编/克制读法、科技收益和结果页复盘已有基线，命令卡已补路线/角色/字母热键执行、程序化 icon 身份、HUD 背包槽、命令状态徽标、冷却/生效 meter、法力缺口 badge 和目标模式高亮；但还需要可配置热键、更强 cursor/目标反馈、发布级 icon art、真实多充能体系、最终 cooldown art 和更少文字依赖。
- 差距雷达只能说明缺口被看见，不能替代真正补齐：当前最硬的 War3 体感缺口仍在真实动作 clips、发布级技能特效、音效、远景剪影、施法/死亡/攻击表现和 T2/T3 战术节奏。

## 第一批协作优先级

1. 先完成 DP-002 资产状态审计，建立真实缺口表。
2. 再完成 DP-001 可读性基线，避免后续模型只追求好看。
3. 然后并行推进 DP-003、DP-005、DP-010，优先覆盖当前五分钟试玩里最常出现的单位和建筑。
