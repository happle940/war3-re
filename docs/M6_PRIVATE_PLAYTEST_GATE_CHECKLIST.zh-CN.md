# M6 Private Playtest Gate Checklist：少量私测准入清单

> 用途：判断某个 `M6` 候选版本是否适合发给少量知情 tester。本文只判断 `private playtest`，不代表可公开分享。

## 1. 私测候选的硬性前置条件

以下任一项为 `No`，结论就是 `M6 private playtest: NO`。

| 检查项 | Yes / No | 证据 |
| --- | --- | --- |
| 用户明确允许少量私测，或至少授权准备私测候选判断。 |  |  |
| 目标候选版本通过 `npm run build`。 |  |  |
| 目标候选版本通过 `npx tsc --noEmit -p tsconfig.app.json`。 |  |  |
| live 链接可直接打开，无白屏、启动崩溃或隐藏本地步骤。 |  |  |
| 初始 HUD 出现，默认镜头下核心对象可见。 |  |  |
| worker 能被选中，右键地面移动和右键采集有可见反馈。 |  |  |
| Town Hall / Barracks 命令卡可读，能完成最小训练闭环。 |  |  |
| 资源扣除、单位出生、采集循环没有破坏性错误。 |  |  |
| AI 不是静止样机，至少能采集、生产或形成基础压力。 |  |  |
| 没有不可恢复卡死、持续刷错、严重性能问题或明显反馈污染。 |  |  |
| Known Issues 已刷新到目标候选版本。 |  |  |
| 私测说明明确写清 alpha 范围、proxy / fallback 视觉、已知限制和反馈边界。 |  |  |
| tester 范围受控，不会把链接当作公开 demo 转发。 |  |  |

## 2. 仍可粗糙但可接受的私测债

下面问题如果已提前披露，可以允许进入少量私测；它们不等于公开分享条件满足：

- 视觉仍是 proxy / fallback，建筑和单位不是最终资产品质。
- `M3` 空间感、镜头和视觉方向仍待用户人工确认。
- `M4` 人机 alpha 质量仍待用户短局判断。
- `M5` 内容范围和视觉身份还没有最终决定。
- 胜负状态已能表达胜利/失败，但缺少重开按钮和赛后总结。
- README / 分享页仍偏工程说明，不够像正式产品入口。
- 只包含有限内容，不包含完整阵营、完整科技树、完整战役或完整 onboarding。
- `M7` 工程硬化仍在进行，但当前可玩路径已有基本合同保护。

这些债的前提是：tester 已知情，且不会因此把反馈集中到“这不像成品”。

## 3. 即使私测也要阻断的问题

出现任一项，不应发送给任何外部 tester：

- 页面打不开、白屏、启动崩溃，或必须靠本地隐藏步骤才能运行。
- 初始 HUD 不出现，或默认镜头下 Town Hall、worker、金矿等核心对象完全不可见。
- worker 无法选中，右键移动 / 采集没有可见反馈。
- Town Hall / Barracks 命令卡不可读，或最小训练闭环无法完成。
- 资源扣除、单位出生、采集循环存在明显破坏性错误。
- AI 完全不行动，tester 只能在空地图里操作。
- 出现不可恢复卡死、持续刷错、严重性能故障。
- HUD、点击、镜头一眼就污染反馈，例如点不中核心对象或 HUD 压住主视图。
- Known Issues 缺失、明显过期，或没有披露会影响理解的限制。
- 私测说明没有交代 alpha 范围、proxy 视觉、非目标和反馈边界。
- 对外文案把当前版本描述成公开 demo、release ready 或完整 Warcraft-like 游戏。
- 用户尚未允许任何外部分享。

## 4. 发送私测链接前必须存在的文档 / 链接

| 文档 / 链接 | 必须状态 |
| --- | --- |
| 试玩链接或候选版本链接 | 指向目标候选版本，不是模糊“当前线上版”。 |
| 私测邀请说明 | 写清这是少量私测，不是公开 demo。 |
| Known Issues | 已同步当前候选版本，披露 proxy、M3/M4/M5/M6 未决点和赛后体验限制。 |
| 反馈路径或反馈模板 | tester 知道如何回报浏览器、设备、核心闭环、AI、阻塞问题和最终判断。 |
| 分享边界 | 明确是否禁止转发，或只允许发给哪些知情 tester。 |
| 停止并报告条件 | 写清白屏、核心交互断裂、AI 完全不动、严重卡死等立即停止项。 |
| M6 red line 复核 | 一级红线没有触发；若不确定，先 hold。 |

## 5. 私测反馈范围

### In-scope feedback

现在最有价值的反馈是：

- 能否正常打开，浏览器 / 设备环境是什么。
- 默认镜头下是否看得懂基地、worker、金矿、树线和 HUD。
- 选择、右键移动、采集、训练是否符合基本 RTS 直觉。
- 资源扣除、单位出生、采集循环是否让人理解。
- AI 是否像一个活着的对手，而不是静止样机。
- HUD / 点击 / 镜头是否误导操作。
- 哪个已知限制最影响继续试玩。
- 这版是否适合继续少量私测，还是应修完再测。

### Out-of-scope feedback

下面反馈可以记录，但不能作为私测 gate 的主要判断：

- 最终美术、单位造型、建筑资产完成度。
- 完整阵营、完整科技树、完整战役或完整新手引导。
- 长期平衡、天梯公平性、正式内容量。
- 是否适合公开传播。
- proxy / fallback 是否像最终产品。
- 发行级胜负包装、重开流程、赛后统计。
- M5 产品方向该如何最终选择。

## 6. Final Private Playtest Gate

只允许一个结论：

```text
M6 private playtest: YES / NO
```

判 `YES` 的最低标准：

- 第 1 节所有硬性前置条件为 `Yes`。
- 第 3 节没有阻断项。
- 第 4 节文档 / 链接齐备。
- tester 是少量、可信、知情的人。
- 反馈边界已经写清，不会被最终美术、完整内容或公开 demo 标准污染。
- 用户允许少量私测，且文案没有暗示公开分享。

任一条不满足，就写：

```text
M6 private playtest: NO
原因：
下一步：
```

`YES` 只表示可以给少量知情 tester。它不表示 `M6` 已批准公开分享，也不表示当前版本可以被宣传为 demo。
