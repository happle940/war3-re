## 2. 英雄等级上限

- 英雄最高等级为 **10**。
- 达到等级 10 后不再获得经验值。

## 6. 普通单位 XP 公式

来源：Blizzard Classic Hero Basics。

基础值：`GrantNormalXP = 25`

递推公式：`F(x) = F(x-1) + 5*x + 5`，其中 x 为单位等级。

| 单位等级 | XP 奖励 |
|---------|---------|
| 1 | 25 |
| 2 | 40 |
| 3 | 60 |
| 4 | 85 |
| 5 | 115 |
| 6 | 150 |
| 7 | 190 |
| 8 | 235 |
| 9 | 285 |
| 10 | 340 |

交叉检查：Liquipedia 给出公式 "Previous level xp gained + 5*(level+1)"，即 `F(x) = F(x-1) + 5*(x+1)`，对等级 1–6 求值为 25/40/60/85/115/150，与主来源公式等价（`5*x + 5 = 5*(x+1)`）。完整序列一致。

## 7. 最小项目映射（HERO10-DATA1 / HERO10-IMPL1）

### 7.1 当前项目可起步范围

- **仅敌方非建筑单位击杀**：当前项目可以从击杀敌方非建筑单位给 Paladin 分配 XP 起步。
- 以下内容 **不在** 起步范围内：
  - 野怪营地（当前项目无野怪）
  - 敌方英雄击杀（当前项目无敌方英雄玩法）
  - 物品经验
  - 光环经验加成
  - 多英雄 XP 分配
  - 盟友经验共享
  - 英雄 UI

### 7.2 建筑排除

- 建筑（非防御塔）击杀 **不** 给予 XP，直到单独的来源/运行时决策做出。
- 当前项目建筑击杀机制未准备承载 War3 XP 语义。

### 7.3 死亡英雄规则

- 已死亡的英雄 **不** 获得经验值。

### 7.4 升级行为

- 达到累计 XP 阈值时 `heroLevel` 增加 1。
- 每次升级授予 **恰好 1** 个 `heroSkillPoints`。
- 升级 **必须** 保留 HERO9 复活公式对 `heroLevel` 的读取（复活费用/时间随等级缩放）。

## 8. 技能点就绪规则

来源：Blizzard Classic Hero Basics。

| 规则 | 描述 |
|------|------|
| 初始技能点 | 每个新英雄起始时拥有 1 个技能点 |
| 升级技能点 | 每次升级授予 1 个新的技能点 |
| 终极技能解锁 | 终极技能在英雄等级 6 时首次可用 |

**延后内容**：能力等级、已学能力 UI、光环、终极技能实现、属性成长。

## 9. HERO9 语义保留

以下 HERO9 闭环语义 **必须** 在 HERO10 实现中保持不变：

| 语义 | 保留要求 |
|------|---------|
| 死亡状态 | 升级不改变 `isDead` 语义；死亡英雄不获得 XP |
| 复活费用按等级 | 升级后 `heroLevel` 变化正确反映在复活费用公式中 |
| 唯一性 | 升级不创建新英雄记录 |
| 无自动选择 | 升级不改变选择逻辑 |
| 圣光合法性 | 升级不改变圣光施放条件 |
| 无重复 Paladin | 升级不绕过唯一性约束 |
