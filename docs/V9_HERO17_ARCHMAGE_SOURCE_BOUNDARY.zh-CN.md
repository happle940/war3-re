# V9 HERO17-SRC1 Archmage 来源边界

> 生成时间：2026-04-17
> 前置：Task261 (HERO17-CONTRACT1) 已 accepted。Archmage 分支边界合同已定义。
> 基线：HERO16 (Paladin AI 全链路收口) 已 accepted。
> 范围：Archmage 英雄单位及其四个能力的来源值与项目映射。**不** 修改生产代码。
> 本文档 **不** 声称"完整英雄系统"、"完整人族"或"V9 发布"。

## 1. 来源层级

| 层级 | 来源 | 角色 |
|------|------|------|
| 主来源 | Blizzard Classic Battle.net Human 页面 (`https://classic.battle.net/war3/human/units/archmage.shtml`) | Archmage 基础属性和能力的优先采纳依据 |
| 主来源 | Blizzard Classic Hero Basics (`https://classic.battle.net/war3/basics/heroes.shtml`) | 英雄通用规则（技能点、终极技能、等级门槛） |
| 交叉检查 | Liquipedia Warcraft III — Archmage (`https://liquipedia.net/warcraft/Archmage`) | 社区维护的当前补丁数值和完整补丁历史 |
| 交叉检查 | Warcraft Wiki (`https://warcraft.wiki.gg`) | 额外交叉校验 |
| 冲突样本 | 旧补丁记录、社区表格、非官方资料 | 只记录差异，不直接采纳 |
| 冲突处理 | 以 Blizzard Classic / Classic 镜像主来源为准；冲突时 Codex 可显式覆盖 | — |

### 1.1 版本选择说明

War3 历经多次补丁（1.03 至 2.0.4），部分数值在不同版本间有变化。本来源边界采用以下原则：

- **基准来源**：采用 `classic.battle.net` 当前可复查页面记录的值作为主源基线；不把记忆中的 RoC / TFT / Reforged 补丁值直接当作生产依据。
- **补丁历史**：记录重大补丁变更和社区样本差异，但采纳值以主来源为准。
- **后续调整**：数值微调可在 runtime 任务中根据游戏体验调整，不在此边界锁定。

| supply | 5 | 一致 | **5** | 无冲突；与 Paladin 同 |
| attackDamage | 21-27 (avg 24) | Liquipedia 一致 | **21** | 采用下限作为当前固定伤害标量；未来若支持伤害范围，另开数值迁移任务 |

| attackType | Hero | Classic 主源明确为 Hero | **Normal** | 项目当前没有 Hero 攻击类型；沿用 Paladin 映射：Hero 攻击 → Normal |
| armorType | Hero | 项目无 Hero 护甲类型 | **Heavy** | 项目映射：Hero 护甲 → Heavy（与 Paladin 一致） |

| 字段 | Paladin 采用值 | Archmage 采用值 | 差异说明 |

| attackDamage | 24 | 21 | Archmage 基础攻击更低 |
| attackRange | 1.0 | 6.0 | Archmage 远程，Paladin 近战 |
| attackCooldown | 2.2 | 2.13 | 接近 |
| maxMana | 255 | 285 | Archmage 法力更多（更高 INT） |
| attackType | Normal | Normal | 两者在主源都是 Hero 攻击；项目暂映射为 Normal |

### 2.3 英雄特有字段

| 字段 | 采用值 | 说明 |
|------|--------|------|
| isHero | true | 英雄标识 |
| heroLevel | 1 | 初始等级 |
| heroXP | 0 | 初始经验 |

### 2.4 来源未知 / 暂缓字段

| 字段 | 状态 | 说明 |
|------|------|------|

### 8.2 Altar of Kings 更新（延后到 HERO17-EXPOSE1）

`HERO17-DATA1` 只允许添加 `UNITS.archmage` 数据，不修改 `BUILDINGS.altar_of_kings.trains`。原因是 `trains` 会直接影响命令卡和训练入口，属于玩家可见暴露面，需要独立 runtime proof。

```

// HERO17-EXPOSE1 才能在 focused runtime proof 下扩展为 ['paladin', 'archmage']
```
